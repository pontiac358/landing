# テーマを素早く作成する

クイックブランディングを使用すると、いくつかのテーマを変更するだけで、すぐに作業を開始したり、後で最適化したりできる結果を得ることができます。いくつかの色を設定するだけで、すべてが必要な場所に、必要なコントロールに適用されます。次に、これをすべてのサポートされているテーマに対して繰り返すだけで完了です。

### 作業を始める前に

1. Figmaライブラリの最新バージョンをダウンロードすることをお勧めします。
2. 値は変数操作インターフェイスで設定されます。Color Stylesコレクションを開き、Brandingグループを選択する必要があります。

   ![Branding](/static/images/design/Branding/Branding.png)

3. 結果はFigmaのBrandingページで確認すると便利です。そこにはブランド変数を使用するコンポーネントが集められており、最終的にどのように見えるかの初期イメージを掴むことができます。
4. コンポーネントではcolorstylesを使用する必要があります。
5. 各colorstyleの値は、対応する変数です。

### 1. 基本的なブランドカラー

**--g-color-base-background**

ページの背景色です。ライトテーマでは白が最もよく使用されますが、他の色を使用することもできます。ダークテーマでは、ニュートラルなグレーの色合いから暖色系や寒色系まで、より幅広い色の選択肢があります。

![background](/static/images/design/Branding/background.png)

**--g-color-base-brand**

主要なブランドカラーです。プライマリボタン、チェックボックス、ラジオボタン、トグルスイッチの背景、およびツールチップのspecialテーマに使用されます。

![Primary brand color](/static/images/design/Branding/PrimaryBrandColor.png)

**--g-color-base-brand-hover**

主要なブランドカラーのホバーカラーです。より暗い色合いを使用しますが、より明るい色や全く異なる色も許容されます。

![Hover](/static/images/design/Branding/Hover.png)

**--g-color-base-brand-selection**

「選択済み」の意味を持つ要素の背景です。

![Selection](/static/images/design/Branding/Selection.png)

**--g-color-base-brand-selection-hover**

選択された要素のホバーカラーです。一般的なタスクではあまり使用されませんが、新しいコンポーネントのすべてのステートを準備する場合やプロトタイプではよく使用されます。

![Selection Hover](/static/images/design/Branding/SelectionHover.png)

**--g-color-line-brand**

ブランドカラーの線です。カードやタブのSelected state、およびスピナーの色に使用されます。

![Lines](/static/images/design/Branding/Lines.png)

**--g-color-text-brand**

背景なしで使用されるブランドカラーのテキストです。

![Text](/static/images/design/Branding/Text.png)

**--g-color-text-brand-heavy**

背景色の上でブランドカラーのテキストを使用します。背景色によるコントラストの低下を補うのに役立ちます。

![Text Heavy](/static/images/design/Branding/Heavy.png)

**--g-color-text-brand-contrast**

主要なコントロールのブランディングカラーを決定したら、それに対してコントラストのある色を選択する必要があります。コントラストは、使用されるすべてのテーマで維持される必要があります。

ブランドカラーがテーマの切り替え時に明るさを大きく変えない場合は、`--g-color-text-light-primary` および `--g-color-text-dark-primary` のような一定の明るさを持つ色を試すことができます。ブランドカラーが明るさを変える場合は、`-g-color-text-primary` または `--g-color-text-inverted-primary` を試すことができます。

![Contrast](/static/images/design/Branding/Contrast.png)

**--g-color-text-link**

リンクの色。

![Link color](/static/images/design/Branding/LinkColor.png)

**--g-color-text-link-hover**

リンクのホバー時の色。

![Hover link color](/static/images/design/Branding/HoverLinkColor.png)

**--g-color-text-link-visited**

訪問済みリンクの色。

![Visited link color](/static/images/design/Branding/VisitedLinkColor.png)

**--g-color-text-link-visited-hover**

訪問済みリンクのホバー時の色。

![Visited link hover color](/static/images/design/Branding/VisitedLinkHoverColor.png)

### 2. タイポグラフィ

すべてのテキストスタイルは、ライブラリの右側の列にある「ローカルスタイル」セクションに記載されています。このセクションは、オブジェクトが選択されていない場合に表示されます（オブジェクトが選択されている場合は、ライブラリではなく選択されたオブジェクトの属性が表示されます）。

![Typography](/static/images/design/Branding/Typography1.png)

テキストスタイルの編集インターフェースにアクセスするには、スタイルの名前の右側にある設定アイコンをクリックします。

![Typography](/static/images/design/Branding/Typography2.png)

スタイルは3つのグループに分かれています。

![Typography](/static/images/design/Branding/Typography3.png)

1.  **Header** — 見出し、サブ見出し、およびディスプレイ用のスタイルが含まれます。
2.  **Text** – 本文テキストおよびキャプション用のスタイルです。
3.  **Misc** – 純粋なコード用、および通常の本文テキストに埋め込まれたコード（インラインスタイル）用のスタイルです。

Header および Text グループでは、1つのフォントを選択できますが、異なるウェイトを設定できます。見出しには、本文フォントよりも高いウェイトが使用されることがよくあります。

Misc グループには、等幅フォントを設定できます。その際、インラインスタイルが本文テキストに埋め込まれた際に正しく表示され、ベースラインがずれないことを確認してください。もしずれる場合は、「行の高さ (Line Height)」パラメータの適切な値を調整する必要があります。

フォントサイズと行の高さをわずかに変更することは許容されます。フォントによって文字の高さと幅は大きく異なる場合があり、場合によってはこの変更を補う必要があります。特定のテキストスタイルが特定のコンポーネントサイズで使用されていることを考慮すると、スタイルのフォントサイズと行の高さを大幅に変更すると、コンポーネントが正しく表示されなくなる可能性があります。したがって、フォントサイズと行の高さを変更する場合は、さまざまなサイズのコンポーネントの表示を確認する必要があります。

### 3. 角丸の半径

このパラメータは、変数操作インターフェイスで設定されます。値は「Border Radius」コレクションにあります。この属性は、プライマリとテクニカルの2つのグループの変数によって設定されます。

![Corner radius](/static/images/design/Branding/CornerRadius.png)

### プライマリセット

この変数のセットは、コード内の変数に対応します。これらの値は、対応するサイズのコントロールに適用されます。

- --g-border-radius-xs: 3px;
- --g-border-radius-s: 5px;
- --g-border-radius-m: 6px;
- --g-border-radius-l: 8px;
- --g-border-radius-xl: 10px;
- --g-border-radius-2xl: 16px.

### テクニカルセット

これは、Check Buttonコンポーネント内の内部要素の角丸半径を計算するためにFigmaでのみ使用されます。これらは「Inner」という接尾辞が付いた変数であり、その値はプライマリセットの変数の値にリンクされている必要があります。Inner変数の値を計算するには、同じサイズのプライマリ変数の値を取り、そこから3を引きます。

たとえば、`--g-border-radius-m: 6px`の場合、`Inner M radius` = 6 - 3 = 3pxとなります。

### 4. テクニカルカラー

このグループには、デザインシステムでサポートされているスクロールバーの色が含まれます。これにより、一部のブラウザのダークテーマでスクロールバーが正しく表示されないことを回避できます。

このグループの変数：

- --g-color-scroll-track;
- --g-color-scroll-handle;
- --g-color-scroll-handle-hover;
- --g-color-scroll-corner.

# 追加のテーマ設定

### プライベートカラーの設定

ブランドカラーに合わせるために特定の色合いを変更する必要がある場合は、プライベートパレットが役立ちます。プライベートカラーは、変数操作インターフェイスの「Private Colors」コレクションで設定されます。

![Private color tuning](/static/images/design/Branding/PrivateColorTuning.png)

デザインガイドソースファイルを日本語に翻訳します。
翻訳はできるだけ自然に聞こえるようにしてください。
これは開発者向けのUI/UXデザインガイドのドキュメントです。
MDXコンポーネントタグ、コードブロック、技術用語はすべてそのまま保持してください。
Button、Alert、Dialogなどのコンポーネント名は翻訳しないでください。
MarkdownコードブロックをHTMLに変換しないでください！

基本のクロマティックカラー：

- 青
- 緑
- 赤
- 黄
- オレンジ
- クールグレー
- 紫

これらのいずれか（またはすべて）をブランドカラーに調整する必要がある場合があります。開発に渡せるパレットを作成するには、[スクリプト](https://preview.gravity-ui.com/uikit/?path=/docs/branding-palette-generator--docs)を使用できます。その結果をFigmaにも転記する必要があります。

ページの背景（または反対のテーマの背景）が変更された場合は、黒と白の新しいソリッドカラーも生成する必要があります。

プライベートパレットが変更されると、セマンティックカラーは自動的に更新されます。

### セマンティックカラーの設定

プライベート変数の値がブランドのビジュアル言語の期待と一致しない場合は、セマンティックカラーの値を変更できます。これらは、Color Stylesコレクションの変数操作インターフェースで設定されます。

![Semantic color tuning](/static/images/design/Branding/SemanticColorTuning.png)

以下を考慮する必要があります。

1. 各テーマに対応するプライベートカラーを使用する必要があります。
2. アルファチャンネルを持つプライベートカラーをソリッドカラーに置き換える（またはその逆）と問題が発生する可能性があります。
3. セマンティック変数を変更すると、それらが使用されているcolorstylesおよびコンポーネントに自動的に適用されます。

# 開発との同期

現時点では、Figmaのライブラリとコード間のテーマの同期は手動で行われています。開発に渡すには、CSS変数を用意する必要があります（[手順を参照](https://preview.gravity-ui.com/uikit/?path=/docs/branding-overview--docs)）。
