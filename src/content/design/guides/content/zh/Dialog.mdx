**Dialog** is used to display secondary content, as well as to focus the user's attention on performing a specific action or scenario. A dialog blocks user interaction with the parent page until it is closed or the target action within the dialog is completed.

### Entry Point

The entry point to a dialog can be a click on a specific trigger (e.g., a button in the interface), or certain actions or conditions that arise in the interface (e.g., attempting to leave the page without saving data, or changing accounts in another tab).

### Sizes

The size of the dialog is chosen based on the content.

![Sizes](/static/images/design/Dialog/sizes.png)

**Size S (480 px)** is used for small warnings or confirmations, as well as small forms.

**Size M (720 px)** is used for small and medium forms.

**Size L (900 px)** is used for large amounts of data and flows within a modal window.

**Custom size** is used when none of the three options (`S`, `M`, or `L`) are suitable.

### Content Area

The content within the dialog can be arbitrary. The layout is chosen based on needs.

If the dialog contains a form, it is recommended to use one of two types of element arrangement:

### Buttons

An action button is almost always needed. If the user has not completed the required actions in the dialog, the button remains in the `Disabled` state.
The left side provides an area for arbitrary controls. Here you can place auxiliary buttons or links, such as help information.

![Group 507124116.png](/static/images/design/Dialog/Group_507124116.png)

有时，可以使用 `danger` 按钮代替 `action` 按钮（例如，在删除实体的对话框中）。

![Group 2087325976.png](/static/images/design/Dialog/Group_2087325976.png)

如果不存在目标操作，并且对话框仅用于信息展示，则不需要 `action` 按钮。在这种情况下，应使用 `Normal` 样式的“关闭”按钮。

![Frame 2087326371.png](/static/images/design/Dialog/Frame_2087326371.png)

`Secondary` 按钮始终用作取消操作。它也可以用于辅助操作（例如，“不更改继续”）。

![Group 2087326024.png](/static/images/design/Dialog/Group_2087326024.png)

对话框中可能完全不包含按钮。通常，这种情况发生在需要在内容区域内进行选择，并且不这样做将导致父界面无法访问时。

![Frame 2087325516.png](/static/images/design/Dialog/Frame_2087325516.png)

### 对话框内导航

如果对话框包含多步流程，则使用“返回”按钮进行导航。

![Group 2087326025.png](/static/images/design/Dialog/Group_2087326025.png)

如果对话框具有嵌套结构，但不是线性步骤，则可以在内容区域使用面包屑导航。

![breadcrumbs](/static/images/design/Dialog/breadcrumbs.png)

### 对话框在页面上的位置

对话框在屏幕上垂直和水平居中对齐。

当视口缩放时，定位会保持不变，对话框的大小会减小，前提是不会影响其内部内容。

与屏幕顶部和左侧边缘的最小允许间距为 20 px。

![position](/static/images/design/Dialog/position.png)

如果对话框无法完全显示在视口内，则顶部和左侧的间距将保持固定，而对话框的右侧和底部将超出屏幕边缘。这种情况是不理想的，建议在设计对话框时，确保它在任何屏幕上都能完全显示在视口内，并且与屏幕边缘的间距至少为 20 px。

### Защита от потери пользовательских данных

Из-за легкости закрытия модального окна есть вероятность сделать это случайно. Если в модальном окне пользователь выбирал только чекбоксы, то потерю таких данных проще восстановить, чем когда пользователь долго писал код или переписывал 12-значный VIN автомобиля. Таких ситуаций мы не должны допустить. При этом нельзя и мешать пользователю выйти из модального окна привычным для него образом.

Чтобы избежать ошибки потери данных, лучше использовать **дополнительное окно подтверждения удаления данных**. В этом случае пользователю нужно будет подтвердить действие.

![Pop-up S.png](/static/images/design/Dialog/Pop-up_S%202.png)

Нужно ли при закрытии модального окна добавлять еще окно подтверждения, — стоит оценивать исходя из здравого смысла. При этом важно соблюсти баланс между безопасностью и возможным раздражением от слишком частых вопросов.  Цель подтверждения — не потерять введенные пользователем данные, не мешая ему при этом выполнять сценарий.
