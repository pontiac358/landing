Кнопка – интерактивный элемент пользовательского интерфейса, по нажатию на который выполняется действие, например: отправка формы, сохранение изменений или переход к следующему шагу. Если необходимо перейти на другую страницу или сайт, рекомендуем использовать ссылку.

## Внешний вид

Существует 5 видов кнопок: акцентные, основные, семантические, приподнятые и контрастные.

### **Акцентные**

![Accent](/static/images/design/Button/Button1.png)

Акцентные кнопки предназначены для выделения ключевых действий и привлечения внимания пользователей к важным операциям. Они представлены тремя основными стилями: Action, Outlined Action и Flat Action.

Выбор между этими типами акцентных кнопок зависит от нужной степени акцента на действиях в интерфейсе. Action подходит для высшего приоритета, Outlined Action — для среднего, а Flat Action — для более низкого приоритета или второстепенных операций.

Рекомендуется осторожно использовать акцентные кнопки в интерфейсе и, по возможности, ограничивать их количество до одной на странице (исключая диалоговые окна). Это помогает подчеркнуть важность и выделить основное действие для пользователя, уменьшая риск перегрузки интерфейса множеством акцентных элементов.

### **Основные**

![Primary](/static/images/design/Button/Button2.png)

Основные кнопки предназначены для выполнения основных и общих действий в интерфейсе. Эти кнопки доступны в нескольких стилях: Normal, Outlined, Flat и Flat Secondary. Выбор между этими стилями зависит от степени акцента на определенных действиях в интерфейсе. Например, Normal-кнопка подходит для обычных действий, Outlined – для действий со средним приоритетом, а Flat и Flat Secondary предназначены для менее важных или второстепенных операций.

### Семантические

Семантические кнопки — это кнопки, которые несут в себе смысловую нагрузку и передают определенный контекст или действие. Они используются для того, чтобы пользователи могли легко понимать, какие операции выполняются и каков их смысл.

Семантические кнопки представлены четырьмя основными стилями: Info, Success, Warning и Danger.

## 结构

按钮在界面中的结构包含多种样式：

![Anatomy](/static/images/design/Button/Button9.png)

1. **Text Only** — A button that contains only text. This type of button is useful when the text itself clearly and unambiguously conveys the button's purpose, especially when the text is short and informative. For example, such a button can be used for "create" or "cancel" actions.
2. **Icon Right** — A button where the icon is located to the right of the text. This option is useful when you need to present different ways to perform the same action. For example, such a button can be used for a "save" action, where the user has several options to choose from ("save as PDF/RTF/HTML...").
3. **Icon Left** — A button where the icon is located to the left of the text. This option is useful when the icon carries key information and needs to be highlighted. For example, such a button can be used for "run" or "delete" actions.
4. **Icon Left and Right** — A button with icons placed to the left and right of the text. This option is useful when you need to emphasize a key action with an icon and provide different paths for its execution. For example, such a button can be used for an "add" action, where the user has several options to choose from ("add project/resource/document...").
5. **Icon Only** — A button that consists only of an icon and does not contain text. This option is useful when the information on the button is easily understood visually and does not require additional text explanations. It is also used when it is necessary to compactly group many actions in a limited area and there is no possibility to place buttons with text labels. For example, a delete button in a list of items can look like an icon with a trash can image without a text label.

## States

Each button in the interface has five main states that help visually indicate the button's availability and current action to the user.

![States](/static/images/design/Button/Button10.png)

**默认** — 按钮的默认状态，此时按钮已准备好进行交互并等待用户操作。

**悬停** — 按钮已准备好进行交互并等待用户操作的状态。

**禁用** — 按钮暂时无法交互的状态。为了提高用户理解，建议解释禁用原因，尤其是在不明显的情况下。此解释可以以按钮旁边的文本形式，或鼠标悬停时的工具提示形式呈现。

**加载中** — 指示系统正在响应用户触发的操作而执行某个操作的状态。

**选中** — 此状态用于可切换元素，用户可以在其中选择多个选项之一。处于 `Selected` 状态的按钮表示当前选中的状态或选项。

## 尺寸

每个按钮都可以使用四种尺寸，允许根据使用场景选择最合适的选项。

![Sizes](/static/images/design/Button/Button11.png)

**XS** — 附加尺寸，用于尺寸受限组件中的操作。

**S** — 应用于普通按钮过大且占用过多空间的情况，例如在表格或小型卡片中。

**M** — 默认推荐使用的主要尺寸。

**L** — 用于在创建表单、模态框或弹出窗口中突出主要操作。

**XL** — 用于需要突出操作且有足够空间放置的场景（例如，在网站或登陆页面上）。

## 形状

`pin` 属性允许通过以下值来控制按钮的形状，定义其样式：

![Form](/static/images/design/Button/Button12.png)

**round** — 圆形按钮（两侧圆角）。这种样式赋予按钮柔和且现代的外观，使其友好且易于交互。

**brick** – 两侧带有尖角的按钮。

**clear** – 按钮两侧带有尖角，且两侧没有边框。

**circle** – 按钮两侧呈圆形。

您还可以组合这些属性以创建不同的变体，从而根据设计理念和界面要求为按钮创建独特的样式。

## 宽度

按钮的默认宽度会自动适应文本，使按钮能够拉伸以容纳所有文本，而不会截断。但是，如果您需要控制按钮的宽度，可以使用两个主要属性：`auto` 和 `max`：

![Width](/static/images/design/Button/Button13.png)

`auto` – 限制按钮的大小，使其不会超出容器。如果按钮内容无法放入容器，它将通过省略号隐藏，表示存在额外文本。

`max` – 将按钮的宽度设置为等于父容器的宽度。如果按钮内容超过此宽度，它也将通过省略号隐藏，从而保持界面的紧凑性并避免容器溢出。

## 建议

**使用**

- 按钮文本应清晰简洁，以便用户立即了解点击后会发生什么。
- 使用图标时，请确保它们清晰地传达其含义，并且用户能够轻松识别它们。
- 在图标和文本之间保持平衡，使两者都可见且易于阅读，并且没有一个会占据主导地位。
- 在仅使用图标的按钮时，我们建议添加悬停提示（`action tooltips`），以便用户了解图标的含义。

**文本撰写**

- 按钮上的文字应清晰简洁地传达点击后将发生的操作。避免使用长句或不明确的缩写。
- 使用描述用户操作的动词，例如“保存”、“发送”、“删除”。
- 避免在按钮上使用带有否定含义的词语或短语。否定指令可能会引起混淆。
- 在整个界面的所有按钮上保持统一的文本风格和格式。这有助于用户轻松导航并理解可用的操作。
