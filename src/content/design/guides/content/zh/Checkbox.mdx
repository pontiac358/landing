复选框是一种界面元素，允许用户从一组选项中选择一个或多个。

## 使用场景

![Cases](/static/images/design/Checkbox/Checkbox1.png)

激活复选框后，操作不会立即执行。要使更改生效，用户需要点击一个确认按钮（例如，“保存”、“应用”、“发送”）。在大多数情况下，复选框会与一个解释其含义的标签一起使用。但是，在某些情况下，复选框可以不带标签使用，例如在表格中，或者当在每个复选框旁边显示“启用”标签显得多余时。文本应以积极的表述开始，例如“启用”而不是“禁用”，否则用户需要解决一个逻辑问题。

## 结构

复选框由以下部分组成：

![Anatomy](/static/images/design/Checkbox/Checkbox2.png)

1.  **容器** - 包围复选框的矩形视觉区域。容器用于指示复选框的边界和激活状态。
2.  **勾选标记** - 当复选框被激活时出现在容器内的符号。此符号作为用户选择该复选框的视觉确认。
3.  **文本标签（可选）** - 位于复选框右侧的文本。文本标签解释了复选框的含义或目的。

## 尺寸

`Checkbox` 组件有两种主要尺寸：

![Sizes](/static/images/design/Checkbox/Checkbox3.png)

**M** - 默认使用，具有适合大多数情况的标准尺寸。在此尺寸下，容器为 14x14 像素，文本标签使用 13 像素的字体。

**L** - 当需要突出复选框与其他界面元素时，或者当界面需要更大的元素以方便使用时使用。在此尺寸下，容器为 17x17 像素，文本标签使用 15 像素的字体。

## 状态

![States](/static/images/design/Checkbox/Checkbox4.png)

**默认** — 复选框的默认状态，点击后可开启或关闭。

**悬停** — 鼠标悬停在复选框区域（复选框 + 标签）时的状态。

**禁用** — 由于某种原因无法与复选框进行交互时的状态。

**不确定** — 复选框组标题的状态，当选中了多个项目中的一个时（常见于多选行的表格中）。

## 建议

### 建议使用

- **选项选择** — 当用户需要从列表中选择一个或多个选项时。

- **功能开启/关闭** — 当用户可以在界面中开启或关闭特定功能时。例如，可以开启或关闭通知、自动播放等功能。在这种情况下，复选框提供了直观的交互方式。

- **表格或列表中的多选** — 当用户需要选择多个项目以进行批量操作（如删除或导出）时。这简化了数据处理，并使批量操作更加便捷。

- **内容或结果过滤** — 在过滤器中，用户选择用于过滤内容或搜索结果的参数时。

### 不建议使用

- **仅选择一个选项** — 如果用户只能从一组互斥的选项中选择一个，则最好使用单选按钮 (radio)。

- **双向开启/关闭（即时操作）** — 如果选项只有两种状态（开启和关闭），并且更改应立即生效，而无需点击“应用”按钮，则最好为此目的使用切换开关 (switches)。它们为此类选项提供了更清晰、更简洁的表示，并允许立即应用更改。

- **Segmented Control** — If there are multiple mutually exclusive options that can be selected simultaneously across different categories, use segment controls. They allow users to select options within different segments or categories.

- **Select** — If there is a long list of options from which to choose one, it is better to use a select. This will save screen space and make it easier to choose from a large set of options.

### Writing Text

- Strive for brevity in text descriptions. Avoid long phrases or complex sentences.
- Use active verbs that clearly indicate the action associated with the selection (e.g., "Subscribe to newsletter" instead of "Newsletter subscription").
- Give users a complete understanding of what will happen if they select a checkbox (e.g., "Receive notifications about new articles" instead of "Notifications").
