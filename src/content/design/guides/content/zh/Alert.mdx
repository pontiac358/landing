Using `Alert` can draw the user's attention and inform them of some important information without interrupting their work.

![Example](/static/images/design/Alert/Example.png)

## When to Use

Primarily, `Alert` is designed to inform about system status, report errors and warnings, and guide the user towards their prompt correction through instructions, links, or commands.

## Types

Depending on the situation, different types of `Alert`s are shown to the user:

- `Danger` is used to inform the user that it is currently impossible to continue working according to the main scenario (e.g., no access to an object) or that further user actions may lead to irreversible loss of data or resources.
- `Warning` is used to inform the user that further work may be difficult or a secondary scenario may be unavailable.
- `Success` is used to inform the user about a status or any change in the system that has a positive connotation (e.g., successful completion of a long operation).
- `Info` is used to provide recommendations for working with the system that should be explicitly shown, or to announce any additional features for the user within the system.
- `Normal` is mainly used to highlight a piece of text, such as a short reference, code example, calculation results, etc.

![Types](/static/images/design/Alert/Types.png)

You should not substitute types solely out of a desire to attract more user attention – for example, displaying important reference information in an `Alert` of type `Warning` or `Error`.

In addition to instructions, `Alert` can also offer a set of commands to reduce the user's effort in finding solutions and taking action.

![Warning](/static/images/design/Alert/Warning.png)

## When Not to Use

`Alert` should be used sparingly because they attract attention and can distract from the main workflow, especially if the `Alert` type is chosen incorrectly.

- 不建议使用 Alert 来展示与表单或对话框中特定字段相关的错误、警告或帮助信息。在这种情况下，最好使用字段组件的 Error 状态，或者在字段旁边以纯文本或带图标的文本形式显示消息。
- 不建议在同一页面（表单、对话框）上放置多个 Danger 和 Warning 类型的 Alert，因为这会增加确定消息优先级的难度。
- 不应将 Alert 用于非预期目的，例如用于视觉突出页面上的某些块：标题、表单字段等。
- 不应使用 Alert 来确认操作的完成。为此，应使用 Dialog。
- 不应使用 Alert 来显示用户发起的某个操作的状态。为此，应使用 Toaster。

## 目录

Alert 中必填项是描述和图标。除此以外，Alert 可选地包含标题 (Title)、关闭按钮 (Close) 和最多三个附加的操作按钮 (Actions)。

### 图标

默认情况下，Alert 会显示与其类型相对应的图标。但是，您可以在组件中定义自己的图标，同时保持背景颜色不变。对于 Normal 类型的 Alert，图标是可选的，默认不显示。

![Icon](/static/images/design/Alert/Icon.png)

### 文本

标题和描述文本应以大写字母开头，并尽可能简短。对于更多信息，最好通过链接或“帮助”、“支持”、“更多”等按钮将用户引导至单独的帮助页面或网站。

标题应简明扼要地概括消息。同时，标题文本不应重复描述。如果无法提炼出主要观点并将其作为标题，最好只保留描述，不设标题。

标题末尾不应包含句点。

### 按钮

根据设计要求和限制，附加按钮可以显示在文本的右侧（Horizontal Layout）或下方（Vertical Layout）。

![Button](/static/images/design/Alert/Button1.png)

## 选项

当按钮数量和名称允许在不换行的情况下为描述和标题提供足够的空间时，可以选择按钮位于右侧的变体。通常，如果 Alert 的宽度小于 500-600 px，最好选择垂直视图。

默认情况下，附加按钮使用 Normal 样式，但根据消息的语义，它们可以是 Action、Contrast-normal 和各种类型的 Outline（Outline-info、Outline-danger 等）。不应将所有三个按钮都设置为不同的样式。Action 按钮只能有一个，同时需要考虑页面的上下文：页面上是否已经有其他 Action 按钮。

![Button](/static/images/design/Alert/Button2.png)

Alert 提供了一个“关闭”按钮，当用户可能希望隐藏消息（因为他们认为消息不再相关）时，应显示此按钮。

![Button](/static/images/design/Alert/Button3.png)

## 变体

Alert 的宽度没有限制，它可以占据页面（表单、对话框）的所有可用空间，或者具有固定的宽度，以匹配主内容的宽度。

如果根据设计，Alert 应紧贴页面（表单、对话框）的边界显示并占据整个宽度（例如，浏览器更新消息或网站 Cookie 存储条款），则 Alert 容器可以不带圆角。

![Options](/static/images/design/Alert/Options1.png)

如果根据设计不需要 Alert 引起用户的积极关注，建议使用带边框的变体（Outlined View），该变体与带填充的变体（Filled View）具有相同的语义类型。

![Options](/static/images/design/Alert/Options2.png)
