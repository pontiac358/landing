使用变更日志，服务团队可以告知用户已发生的更改。

变更日志在首次打开服务时显示一次。无法重复打开变更日志；如有需要，您可以自行实现此功能。

由于变更日志显示在模态窗口中，因此它遵循适用于模态窗口的关闭和缩放规则。

## 元素

变更日志是一个 `M` 尺寸的模态窗口，其中按时间顺序排列了帖子。每个帖子包含：

1. 发布日期；
2. `New` [标签](https://gravity-ui.com/design/guides/label)（可选）；
3. 标题；
4. 图片（可选）；
5. 文本；
6. 操作 [按钮](https://gravity-ui.com/design/guides/button)（可选）。

![Elements](/static/images/design/ChangelogDialog/ChangelogDialog1.png)

### 发布日期

通过发布日期，用户可以了解服务中何时发生了特定更改。默认情况下，日期格式为 `DD Mon YYYY`，但也可以根据服务的本地化使用其他日期格式。

### 标签

`New` 标签用于标记新帖子。哪些帖子被视为新帖子并用此标签标记，由服务决定。也可以使用不同于 `New` 的其他文本表述。

### 标题

帖子的标题用于简要描述服务中的更改。标题的长度不限。不建议：

- 使用超过一行的标题（超过 70 个字符），因为这会降低文本的可读性；
- 在标题中重复新闻文本。

标题使用 `text-subheader-3` 样式。不提供使用其他文本样式。

### 图片

Here's the translated design guide source file to Chinese:

图片是可选的，尺寸固定为 516x240 像素。可以使用 JPG 和 PNG 格式的图片文件；不支持 GIF 图片和视频文件。当需要说明帖子主题时使用图片。浅色和深色界面主题使用同一张图片。

如果上传的图片尺寸超过 516x240 像素，它将被缩放并居中对齐较长的一边。

考虑到 Retina 显示屏，建议将图片尺寸放大两倍，即 1032x480 像素。

### 文本

图片下方是帖子的文本，长度可任意。不建议：

- 使用过长的文本（超过 500 个字符）。如果无法将文本缩短到建议长度，可以在文本中添加指向文档的链接，或通过点击文本后的按钮跳转到帖子的完整内容。
- 创建副标题。由于建议的文本长度不需要过长，因此不需要使用副标题进行额外的分块。

文本使用 `text-body-1` 样式。不提供使用其他文本样式，但支持格式化：创建项目符号列表和编号列表，文本删除线和下划线，以及粗体和斜体。

### 操作按钮

操作按钮是可选的，位于帖子文本下方。用于跳转到帖子的完整内容。

只允许使用 `M` 尺寸的按钮。按钮可以是任何类型，带图标或不带图标。建议使用同一种类型的按钮。

不支持多个按钮。

### 其他元素

在模态框标题下方，还可以有一个 `View full changelog` 链接，指向 changelog 页面；这可以是服务的独立页面、storybook、第三方文档服务等。

## 使用建议

## Changelog

Changelogs are applied:

- When it's necessary to inform the user about significant changes in the service.
- When it's necessary to keep the user updated on minor changes. In this case, it's advisable to show the changelog periodically and report all minor changes since the last entry.
- When you want to remind the user about changes in the service and increase their awareness of previous releases.

It is not recommended:

- To use changelogs for onboarding.
- To use changelogs to inform the user about every minor change in the service.
