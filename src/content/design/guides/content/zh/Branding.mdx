# 快速创建主题

快速品牌化允许您更改几个主题并立即获得可用的结果，您可以立即开始使用或稍后进行优化。为此，您只需设置几种颜色，所有内容都会应用到正确位置的正确控件上。然后，您需要为所有支持的主题重复此步骤，即可完成。

### 开始之前

1. 我们建议下载最新版本的 Figma 库。
2. 值在变量操作界面中进行设置。您需要打开 Color Styles 集合并选择 Branding 组：

   ![Branding](/static/images/design/Branding/Branding.png)

3. 在 Figma 的 Branding 页面上查看结果非常方便。该页面收集了使用品牌变量的组件，这将帮助您初步了解最终的外观。
4. 在组件中必须使用 color styles。
5. 每个 color style 的值是相应的变量。

### 1. 基本品牌颜色

**--g-color-base-background**

页面背景色。浅色主题通常使用白色，但也可以使用任何其他颜色。对于深色主题，颜色选择更广泛：从中性灰色调到暖色或冷色。

![background](/static/images/design/Branding/background.png)

**--g-color-base-brand**

主要品牌颜色。主按钮、复选框、单选按钮、开关的背景色，以及工具提示的 special 主题。

![Primary brand color](/static/images/design/Branding/PrimaryBrandColor.png)

**--g-color-base-brand-hover**

主品牌颜色的悬停颜色。我们使用更深的色调，但允许使用更浅的颜色或完全不同的颜色。

![Hover](/static/images/design/Branding/Hover.png)

**--g-color-base-brand-selection**

带有“选中”语义的元素的背景色：

![Selection](/static/images/design/Branding/Selection.png)

**--g-color-base-brand-selection-hover**

选中元素的悬停颜色。在典型任务中不经常使用，更多地用于为新组件准备所有状态或在原型中。

![Selection Hover](/static/images/design/Branding/SelectionHover.png)

**--g-color-line-brand**

品牌颜色的线条。卡片和标签中的 Selected 状态，以及加载指示器的颜色。

![Lines](/static/images/design/Branding/Lines.png)

**--g-color-text-brand**

不带背景使用的品牌颜色文本。

![Text](/static/images/design/Branding/Text.png)

**--g-color-text-brand-heavy**

用于在背景上显示品牌文本。有助于补偿背景色带来的对比度下降。

![Text Heavy](/static/images/design/Branding/Heavy.png)

**--g-color-text-brand-contrast**

在确定了主要控件的品牌颜色后，需要为其选择一个对比色。在所有使用的主题中都应保持对比度。

如果品牌颜色在切换主题时亮度基本不变，可以尝试使用亮度恒定的颜色：`--g-color-text-light-primary` 和 `--g-color-text-dark-primary`。如果品牌颜色亮度会发生变化，可以尝试使用 `--g-color-text-primary` 或 `--g-color-text-inverted-primary`。

![Contrast](/static/images/design/Branding/Contrast.png)

**--g-color-text-link**

链接颜色。

![Link color](/static/images/design/Branding/LinkColor.png)

**--g-color-text-link-hover**

链接悬停时的颜色。

![Hover link color](/static/images/design/Branding/HoverLinkColor.png)

**--g-color-text-link-visited**

已访问链接的颜色。

![Visited link color](/static/images/design/Branding/VisitedLinkColor.png)

**--g-color-text-link-visited-hover**

已访问链接悬停时的颜色。

![Visited link hover color](/static/images/design/Branding/VisitedLinkHoverColor.png)

### 2. 字体排印

所有文本样式都列在库的右侧栏的“Local Styles”部分。如果没有选择任何对象，则会显示此部分（否则会显示所选对象的属性，而不是库的属性）：

![Typography](/static/images/design/Branding/Typography1.png)

要访问文本样式的编辑界面，请点击样式名称右侧的设置图标：

![Typography](/static/images/design/Branding/Typography2.png)

样式分为 3 组。

![Typography](/static/images/design/Branding/Typography3.png)

1.  **Header** — 包含标题、副标题和显示样式。
2.  **Text** – 用于正文和标签的样式。
3.  **Misc** – 用于纯代码以及嵌入到普通正文中的代码（内联样式）的样式。

对于 Header 和 Text 组，可以选择一种字体，但设置不同的字重。标题通常比正文字体使用更高的字重。

对于 Misc 组，可以设置等宽字体。同时，应确保内联样式在嵌入到正文中时能正确显示，并且基线不会发生偏移。如果不是这样，则应选择合适的行高（Line Height）参数值。

在样式方面，允许对字号 (Font Size) 和行高 (Line Height) 参数进行微调。在不同的字体中，字符的高度和宽度可能差异很大，在某些情况下需要进行补偿。请注意，特定的文本样式用于特定尺寸的组件，如果样式参数发生显著变化，组件可能会显示不正确。因此，在更改字号 (Font Size) 和行高 (Line Height) 时，应检查组件在不同尺寸下的显示效果。

### 3. 圆角半径

此参数在变量操作界面中进行设置。值位于“圆角半径 (Border Radius)”集合中。此属性由两组变量定义——主变量和技术变量：

![Corner radius](/static/images/design/Branding/CornerRadius.png)

### 主集

此变量集对应于代码中的变量。这些值将用于相应尺寸的控件：

- --g-border-radius-xs: 3px;
- --g-border-radius-s: 5px;
- --g-border-radius-m: 6px;
- --g-border-radius-l: 8px;
- --g-border-radius-xl: 10px;
- --g-border-radius-2xl: 16px.

### 技术集

仅在 Figma 中使用，用于计算 Check Button 组件内部元素的圆角半径。这些变量带有“Inner”后缀，其值应绑定到主集变量的相应值。要计算“Inner”变量的值，请取相同尺寸的主变量值并减去 3。

例如，如果 `--g-border-radius-m: 6px`，则 `Inner M radius` = 6 - 3 = 3px。

### 4. 技术颜色

此组包括设计系统中支持的滚动条颜色。设置这些颜色是为了避免在某些浏览器中，滚动条在深色主题下显示不正确。

此组中的变量：

- --g-color-scroll-track;
- --g-color-scroll-handle;
- --g-color-scroll-handle-hover;
- --g-color-scroll-corner.

# 主题附加设置

### 私有颜色设置

如果需要更改特定颜色的色调，以使其符合品牌颜色，则可以使用私有调色板。私有颜色在变量操作界面中的“私有颜色 (Private Colors)”集合中进行设置。

![Private color tuning](/static/images/design/Branding/PrivateColorTuning.png)

基础色：

- 蓝色；
- 绿色；
- 红色；
- 黄色；
- 橙色；
- 冷灰色；
- 紫色。

以上任何一种颜色（或所有颜色）都可能需要调整为品牌色。要创建一个可供开发使用的调色板，可以使用 [脚本](https://preview.gravity-ui.com/uikit/?path=/docs/branding-palette-generator--docs)，其结果需要同时迁移到 Figma 中。

如果更改了页面背景（或深色主题下的背景），也应生成新的纯色黑色和白色。

更改私有调色板后，语义化颜色将自动更新。

### 设置语义化颜色

如果私有变量的值与品牌视觉语言的预期不符，可以更改语义化颜色的值。它们在 Color Styles 集合的变量操作界面中进行设置。

![Semantic color tuning](/static/images/design/Branding/SemanticColorTuning.png)

需要注意以下几点：

1. 应为相应主题使用私有颜色。
2. 用纯色替换带 alpha 通道的私有颜色（反之亦然）可能会导致问题。
3. 更改语义化变量将自动应用于使用它们的 color styles 和组件。

# 与开发同步

目前，Figma 库和代码之间的主题同步是手动进行的。要将其提供给开发人员，需要准备 CSS 变量（[参见说明](https://preview.gravity-ui.com/uikit/?path=/docs/branding-overview--docs)）。
