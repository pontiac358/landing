**List** — универсальный компонент списка, представляющий собой контейнер с элементами данных, расположенными в один вертикальный столбец. Компонент позволяет сортировать дочерние элементы `list-item` с поддержкой различных размеров, а также управлять их фильтрацией и выделением с помощью клавиатуры.

# Размер

![65.png](/static/images/design/List/65.png)

**Элемент списка (`List-item`)**

Все элементы списка поддерживают отрисовку в 4 размерах: `S`, `M`, `L` и `XL`. Размеры и отступы соответствуют элементам библиотеки, с которыми взаимодействует компонент, — например, `select`, `input`, `button`, которые поддерживают аналогичные переменные размерности.

![63.png](/static/images/design/List/63.png)

**Список (`List`)**

По умолчанию ширина компонента `List` динамическая и рассчитывается по ширине самого длинного элемента в списке. Опционально ширина может быть задана фиксированным значением или наследоваться от объекта, который вызвал отображение списка (`select`, `input`).

# `List-item` Features

![59.png](/static/images/design/List/59.png)

**Multiple Icon Support**

All **list-item** elements support displaying one or more icons on the left and right edges of the component.

![64.png](/static/images/design/List/64.png)

**Sorting Order Change**

Sorting can be enabled for list items. By default, the sorting icon is always displayed at the beginning of the list.

# `List` Features

![66.png](/static/images/design/List/66.png)

**Customizable Height for the Overall List Container**

The height of list items or a function that returns the height value can be set dynamically or specified with an exact value in px. If the list items do not fit within the height, a scrollbar appears. The height can be set manually or changed dynamically.

![70.png](/static/images/design/List/70.png)

**Single and Multiple Selection**

The component supports both single and multiple item selection.

![68.png](/static/images/design/List/68.png)

**Filtering and Search**

An input field can optionally be enabled in the component for quick filtering of items within the list. The default filter placeholder can be changed depending on the component's purpose.

![69.png](/static/images/design/List/69.png)

**Triggering Calls**

By default, the `List` component is triggered by interaction with other elements on the page, such as `Select`, `Label`, `Button`, and is also used as a suggestion for the `Input` component.

![67.png](/static/images/design/List/67.png)

**Virtualization**

With the virtualization flag enabled, the browser renders only a small subset of rows at a given time, significantly reducing the time required for re-rendering components. When using this method, commonly referred to as "infinite scrolling," new items are added to the list as the user scrolls to a specific threshold element (trigger).
