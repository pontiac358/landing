# 버튼

버튼은 사용자 인터페이스의 대화형 요소로, 클릭 시 양식 제출, 변경 사항 저장 또는 다음 단계로 이동과 같은 작업을 수행합니다. 다른 페이지나 웹사이트로 이동해야 하는 경우 링크를 사용하는 것이 좋습니다.

## 모양

버튼에는 액센트, 기본, 의미론적, 돌출형 및 대비형의 5가지 유형이 있습니다.

### **액센트**

![Accent](/static/images/design/Button/Button1.png)

액센트 버튼은 주요 작업을 강조하고 사용자에게 중요한 작업에 주의를 기울이도록 설계되었습니다. 액센트 버튼은 Action, Outlined Action 및 Flat Action의 세 가지 주요 스타일로 제공됩니다.

이러한 액센트 버튼 유형 간의 선택은 인터페이스에서 작업에 필요한 강조 수준에 따라 달라집니다. Action은 최우선 순위에 적합하고, Outlined Action은 중간 우선 순위에, Flat Action은 우선 순위가 낮거나 보조 작업에 적합합니다.

인터페이스에서 액센트 버튼을 신중하게 사용하고 가능한 한 페이지당 하나로 제한하는 것이 좋습니다(대화 상자 제외). 이렇게 하면 사용자를 위한 주요 작업을 강조하고 명확히 하는 데 도움이 되며, 너무 많은 액센트 요소로 인해 인터페이스가 과부하될 위험을 줄입니다.

### **기본**

![Primary](/static/images/design/Button/Button2.png)

기본 버튼은 인터페이스에서 기본적이고 일반적인 작업을 수행하도록 설계되었습니다. 이러한 버튼은 Normal, Outlined, Flat 및 Flat Secondary의 여러 스타일로 제공됩니다. 이러한 스타일 간의 선택은 인터페이스에서 특정 작업에 대한 강조 수준에 따라 달라집니다. 예를 들어, Normal 버튼은 일반 작업에 적합하고, Outlined는 중간 우선 순위 작업에 적합하며, Flat 및 Flat Secondary는 덜 중요하거나 보조 작업에 사용됩니다.

### 의미론적

의미론적 버튼은 의미론적 의미를 전달하고 특정 컨텍스트 또는 작업을 나타내는 버튼입니다. 사용자가 어떤 작업이 수행되고 그 의미가 무엇인지 쉽게 이해할 수 있도록 합니다.

의미론적 버튼은 Info, Success, Warning 및 Danger의 네 가지 주요 스타일로 제공됩니다.

### Info

![Info](/static/images/design/Button/Button3.png)

추가 정보, 팁 또는 컨텍스트와 관련된 작업에 사용됩니다. 정보 버튼은 다른 리소스로 이동할 수 있음을 강조해야 할 때 하이퍼링크 역할을 할 수도 있습니다(예: "더 보기", "문서", "도움말").

**Success**

![Success](/static/images/design/Button/Button4.png)

인터페이스에서 성공적으로 완료된 작업을 강조하는 데 사용됩니다.

**Warning**

![Warning](/static/images/design/Button/Button5.png)

잠재적 위험 또는 경고로 인해 사용자 주의가 필요한 작업을 강조하는 데 사용됩니다. 이러한 버튼은 사용자가 작업의 가능한 결과를 인지하고 더 주의하도록 돕습니다.

**Danger**

![Danger](/static/images/design/Button/Button6.png)

실행 시 파괴적이거나 부정적인 결과를 초래할 수 있는 작업을 강조하는 데 사용됩니다. 잠재적으로 부정적인 측면(예: 삭제, 중지, 비활성화)을 강조해야 하는 경우에만 주의해서 사용하고 실제로 필요한 경우에만 사용하십시오.

**Raised**

![Raised](/static/images/design/Button/Button7.png)

Raised 버튼은 콘텐츠 위에 돌출된 버튼 유형입니다. 일반적으로 고정된 위치에서 화면의 중요한 작업을 강조하는 데 사용됩니다. 이 버튼 유형은 사용자가 즉시 알아차려야 하는 주요 작업에 적합합니다.

이러한 버튼은 카드 인터페이스 및 그래픽 애플리케이션에서도 확대/축소, 이동 또는 기타 그래픽 데이터 조작과 같은 다양한 작업을 수행하는 데 자주 사용됩니다.

**Contrast Buttons**

![Contrast](/static/images/design/Button/Button8.png)

**Contrast Buttons**는 복잡한 배경에서 작업을 강조하는 데 사용되는 특수 버튼 유형입니다. 이러한 버튼은 배너 및 마케팅 자료와 같이 다양하고 풍부한 배경에서도 뛰어난 가시성을 보장하는 밝고 명확한 대비를 만듭니다.

이 버튼들은 액션을 강조하고 배너와 같은 다양한 배경에서도 잘 보이도록 도와줍니다.

## 구조

UI에서 버튼의 구조는 다양한 스타일 옵션을 포함합니다.

![Anatomy](/static/images/design/Button/Button9.png)

1. **Text only** — A button that contains only text. This type of button is useful when the text itself clearly and unambiguously conveys the button's purpose, especially when the text is short and informative. For example, such a button can be used for "create" or "cancel" actions.
2. **Icon on the right** — A button where the icon is located to the right of the text. This option is useful when you need to present different ways to perform the same action. For example, such a button can be used for a "save" action, where the user has several options to choose from ("save as PDF/RTF/HTML...").
3. **Icon on the left** — A button where the icon is located to the left of the text. This option is useful when the icon carries key information and needs to be highlighted. For example, such a button can be used for "run" or "delete" actions.
4. **Icons on the right and left** — A button with icons placed on both the left and right of the text. This option is useful when you need to emphasize a key action with an icon and provide different paths for its execution. For example, such a button can be used for an "add" action, where the user has several options to choose from ("add project/resource/document...").
5. **Icon only** — A button that consists only of an icon and does not contain text. This option is useful when the information on the button is easily understood visually and does not require additional text explanations. It is also used when it is necessary to compactly group many actions in a limited area and there is no possibility to place buttons with text labels. For example, a delete button in a list of items can look like an icon with a trash can image without a text label.

## States

Each button in the interface has five main states that help visually indicate the button's availability and current action to the user.

![States](/static/images/design/Button/Button10.png)

**기본** — 버튼의 기본 상태로, 상호작용할 준비가 되어 있으며 사용자 입력을 기다립니다.

**호버** — 버튼이 상호작용할 준비가 되어 있으며 사용자 입력을 기다리는 상태입니다.

**비활성화** — 버튼을 일시적으로 상호작용할 수 없는 상태입니다. 사용자의 이해를 돕기 위해 비활성화 이유를 설명하는 것이 좋습니다. 특히 명확하지 않은 경우 더욱 그렇습니다. 이 설명은 버튼 옆 텍스트 또는 마우스 호버 시 툴팁으로 제공될 수 있습니다.

**로딩** — 사용자가 시작한 작업에 대한 응답으로 시스템이 작업을 수행 중임을 나타내는 상태입니다.

**선택됨** — 사용자가 여러 옵션 중에서 하나를 선택할 수 있는 토글 가능한 요소에 사용되는 상태입니다. `선택됨` 상태의 버튼은 현재 선택된 상태 또는 옵션을 나타냅니다.

## 크기

각 버튼은 네 가지 크기로 사용할 수 있으며, 사용 맥락에 따라 가장 적합한 옵션을 선택할 수 있습니다.

![Sizes](/static/images/design/Button/Button11.png)

**XS** — 크기가 제한된 컴포넌트 내의 작업에 사용되는 추가 크기입니다.

**S** — 일반 버튼이 너무 크고 많은 공간을 차지하는 상황(예: 테이블 또는 작은 카드)에 적용됩니다.

**M** — 기본적으로 사용을 권장하는 기본 크기입니다.

**L** — 생성 양식, 모달 또는 팝업 창에서 주요 작업을 강조하는 데 사용됩니다.

**XL** — 작업을 강조해야 하고 배치할 충분한 공간이 있는 곳(예: 웹사이트 또는 랜딩 페이지)에 사용됩니다.

## 모양

`pin` 속성을 사용하면 버튼의 모양을 제어하고 다음 값으로 스타일을 정의할 수 있습니다.

![Form](/static/images/design/Button/Button12.png)

**round** — 버튼이 둥근 모양(양쪽 모서리가 둥글게 처리됨)입니다. 이 스타일은 버튼에 부드럽고 현대적인 느낌을 주어 친근하고 상호작용하기 좋게 만듭니다.

**brick** – 양쪽 끝이 날카로운 모서리를 가진 버튼입니다.

**clear** – 버튼은 날카로운 모서리를 가지며 양쪽 끝에 테두리가 없습니다.

**circle** – 버튼은 양쪽 끝이 원형입니다.

이러한 속성을 다양한 조합으로 활용하여 디자인 컨셉과 인터페이스 요구 사항에 따라 고유한 버튼 스타일을 만들 수 있습니다.

## 너비

버튼의 기본 너비는 텍스트에 맞춰 자동으로 조정되므로 버튼이 잘리지 않고 전체 텍스트를 수용할 수 있도록 너비가 늘어납니다. 그러나 버튼 너비를 제어해야 하는 경우 `auto`와 `max`의 두 가지 주요 속성을 사용할 수 있습니다.

![Width](/static/images/design/Button/Button13.png)

`auto` – 버튼 크기를 제한하여 컨테이너를 벗어나지 않도록 합니다. 버튼 내용이 컨테이너에 맞지 않으면 추가 텍스트가 있음을 나타내는 말줄임표로 숨겨집니다.

`max` – 버튼의 고정 너비를 부모 컨테이너의 너비와 같게 설정합니다. 버튼 내용이 이 너비를 초과하면 말줄임표로 숨겨져 인터페이스의 컴팩트한 모양을 유지하고 컨테이너 오버플로우를 방지합니다.

## 권장 사항

**사용법**

- 사용자가 클릭 후 어떤 일이 발생할지 즉시 이해할 수 있도록 버튼 텍스트를 명확하고 간결하게 작성하세요.
- 아이콘을 사용할 때는 아이콘이 의미를 명확하게 전달하고 사용자가 쉽게 인식할 수 있는지 확인하세요.
- 아이콘과 텍스트 간의 균형을 유지하여 두 요소 모두 눈에 잘 띄고 읽기 쉬우며 어느 한쪽이 지배적이지 않도록 하세요.
- 아이콘만 있는 버튼을 사용할 때는 사용자가 아이콘의 의미를 이해할 수 있도록 툴팁(`action tooltips`)을 추가하는 것이 좋습니다.

## 디자인 가이드 소스 파일 (ko)

이것은 개발자를 위한 UI/UX 디자인 가이드 문서입니다.

---

- 버튼의 텍스트는 클릭 후 발생할 동작의 핵심을 명확하고 간결하게 전달해야 합니다. 긴 문구나 이해하기 어려운 약어는 피하세요.
- "저장", "보내기", "삭제"와 같이 사용자가 수행하는 동작을 설명하는 동사를 사용하세요.
- 버튼에 부정적인 단어나 문구를 사용하는 것을 피하세요. 부정적인 지시는 혼란을 야기할 수 있습니다.
- 인터페이스의 모든 버튼에 걸쳐 일관된 스타일과 텍스트 작성 형식을 유지하세요. 이는 사용자가 어떤 동작이 가능한지 쉽게 파악하고 이해하는 데 도움이 됩니다.
