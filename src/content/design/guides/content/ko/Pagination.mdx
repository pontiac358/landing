컴포넌트를 사용하면 사용자가 콘텐츠를 페이지별로 볼 수 있습니다. 예를 들어 테이블에서 사용할 수 있습니다.

![IMG - 8.png](/static/images/design/Pagination/paginator-0001.png)

## 사용 사례

✅ **언제 사용해야 할까요?**

- 페이지네이션이 필요한 모든 경우.
- 여러 페이지를 건너뛰거나 원하는 페이지로 바로 이동해야 할 수 있는 경우.
- 한 페이지에 표시할 행 수를 조정하는 것이 유용한 경우.
- 페이지 내 블록 크기를 고정하여 상단 콘텐츠의 자동 로딩이 하단 콘텐츠를 지속적으로 밀어내지 않도록 하는 경우.

![3.png](/static/images/design/Pagination/paginator-0002.png)

🚫 **언제 사용하지 말아야 할까요?**

결과 페이지 간의 원활한 전환이 중요한 경우에는 자동 로딩 또는 버튼을 통한 로딩을 사용하는 것이 좋습니다. 단, 이로 인해 페이지 성능이 저하되지 않아야 합니다.

## 구조

![IMG - 18.png](/static/images/design/Pagination/paginator-0005.png)

1. **현재 페이지** — 표시되는 페이지의 순번을 나타냅니다.
2. **이전 페이지** — 이전 페이지를 엽니다.
3. **다음 페이지** — 다음 페이지를 엽니다.
4. **마지막 콘텐츠 페이지** — 전체 콘텐츠 양을 알고 페이지로 나눌 수 있는 경우 표시됩니다. 클릭하면 마지막 페이지를 엽니다.
5. **오버플로우 아이콘** — 페이지가 4개보다 많을 때 나타납니다.
6. **앞으로/뒤로 이동 버튼** — 페이지를 순서대로 전환합니다.
7. **첫 페이지로 이동** — 첫 번째 콘텐츠 페이지를 엽니다.
8. **특정 페이지로 이동** — 페이지 번호를 입력하고 Enter 키를 누르면 해당 페이지를 엽니다.
9. **페이지당 항목 수** — 한 페이지에 표시되는 항목 수를 전환하는 컨트롤입니다.

구조의 모든 요소를 함께 사용할 필요는 없습니다. 사용 가능한 구성은 "수정" 섹션에 나열되어 있습니다.

## 크기

기본적으로 페이지네이터의 버튼, 입력 필드, 선택 상자의 크기는 `M`입니다.

## 상태

![IMG - 19.png](/static/images/design/Pagination/paginator-0007.png)

**첫 페이지** — 첫 페이지를 표시합니다.

**두 번째/다음 페이지** — 두 번째 페이지 및 이후 페이지를 표시하는 상태입니다.

**마지막 페이지** — 마지막 페이지를 표시합니다.

## 수정

![IMG - 20.png](/static/images/design/Pagination/paginator-0008.png)

1. **페이지 크기 옵션 및 입력 표시** — 특정 페이지로 이동하거나 페이지당 표시되는 콘텐츠 수를 관리할 수 없는 경우 이러한 필드를 비활성화할 수 있습니다.
2. **총계 알 수 없음** — 현재 페이지만 표시하는 옵션입니다. 총 항목 수를 알 수 없을 때 사용됩니다.
3. **페이지 숨기기** — 페이지 번호 없이 페이지 전환을 표시합니다. 항목 목록이 동적이고 페이지 수와 페이지 번호가 계속 변경되어 사용자를 혼란스럽게 할 수 있는 경우에 사용됩니다.
4. **컴팩트** — 텍스트가 없는 컴팩트한 옵션입니다. 인터페이스 영역이 제한적인 경우(예: 대화 상자)에 사용됩니다.

모바일 장치 인터페이스의 경우 모바일 버전 페이지네이터를 사용할 수 있습니다.
