Indicates the progress of a task: loading, uploading data, processing, etc., until the interface is fully ready for user interaction. The indicator does not show measurable progress, but only indicates that the process is not yet complete and is ongoing.

### **Usage**

âœ…Â **When to use**

1. When loading takes more than 300 ms. `Loader` should not appear for very short periods, as this leads to an unpleasant flicker. If you show a `Loader`, avoid the flickering effect. If loading takes less than 300 ms, show nothing.
2. When the appearance of content after loading cannot be predicted.
3. If `Loader` is used for asynchronous loading, it should be displayed exactly where the loading is happening. Loading a single element should not block the rest of the interface.

1. If loading takes more than 10 seconds and we know how long it will take, it's better to show a progress bar.
2. If the appearance of content can be predicted, a skeleton might be suitable for such cases, and it makes sense to use it.
3. If loading occurs due to a button click, the state of the button itself will be sufficient for feedback, for example, in modal windows.
