레이블은 객체에 대한 텍스트 정보를 포함하며, 상태나 유형과 같은 속성을 설명합니다. 레이블은 범주화, 표시 또는 분류에 사용되며, "키-값" 형식으로 엔티티를 표시하는 데도 사용됩니다.

### 사용 사례

레이블은 어디에나 사용됩니다(예: 카드, 테이블, 엔티티 페이지). 객체의 속성을 나타내며 사용자가 콘텐츠를 빠르게 식별하고 분류하는 데 도움이 됩니다. 일반적으로 레이블은 관련 객체 옆에 표시됩니다.

또 다른 사용 사례는 목록에 맞지 않는 객체 수를 표시하는 것입니다. 이 경우 레이블에 마우스를 올리면 나머지 엔티티 목록이 포함된 팝오버가 나타납니다.

![DLQ](/static/images/design/Label/DLQ1.png)

레이블은 엔티티를 정렬하고 더 편리하게 작업하기 위한 태그 역할을 할 수도 있습니다.

![DLQ](/static/images/design/Label/DLQ2.png)

테이블에 레이블을 표시할 때는 셀당 레이블 수를 제한해야 합니다(연속으로 최대 2-3개 레이블, 테이블당 최대 2개의 레이블 열). 또한 문자 수를 제한해야 합니다(레이블당 최대 20자, 그 이상은 말줄임표).

### 구조

![Structure](/static/images/design/Label/Structure2.png)

![Structure](/static/images/design/Label/Structure.png)

![Application Examples](/static/images/design/Label/Application.png)

1. 왼쪽 아이콘: 선택 사항이며 상호작용하지 않습니다. 정보를 더 빠르게 읽는 데 사용됩니다.

2. 텍스트: 레이블당 20자 이하로 사용하는 것이 좋습니다. 형용사("새로", "활성", "무료"), 분사("중지됨"), 명사("할인") 및 부사를 사용할 수 있습니다. 레이블 텍스트에는 동사가 포함되어서는 안 됩니다(예: "켜기"). 동사가 필요한 경우 버튼을 사용하는 것이 좋습니다.

3. "키-값" 형식의 대체 텍스트 옵션으로, 사용자가 객체를 수동으로 태그하는 데 사용됩니다. 예를 들어 객체에 `service: delivery`, `release: 23`과 같은 태그를 지정할 수 있습니다.

4 — 오른쪽 아이콘, 인터랙티브. 닫기 또는 복사 아이콘 등으로 사용할 수 있습니다.

### 크기

![Sizes](/static/images/design/Label/Sizes.png)

**레이블은 3가지 크기가 있습니다:**

**XS** — 대부분의 경우에 사용됩니다: 테이블, 목록, 제목, 팝업, 엔티티 페이지.

**S** — 폼에서 추가된 레이블 미리보기에 사용되며, 공간이 허용하는 경우에 사용됩니다.

**M** — 버튼과 유사해 보이기 때문에 자주 사용되지 않는 크기입니다.

### 테마

![Themes](/static/images/design/Label/Themes.png)

![Themes](/static/images/design/Label/Themes2.png)

**의미론적 색상 없음**:

**Default** — 기본 레이블, 의미론적 색상 없이 정보를 강조하는 데 자주 사용됩니다.

**Clear** — 배경 없이 테두리만 있습니다. 최소한의 강조에 사용됩니다.

**의미론적 색상 있음**:

**Unknown** — 중립적인 옵션, 레이블로 감싸인 정보에 주의를 집중할 필요가 없을 때 사용됩니다.

**Accent** — 중요한 정보를 강조하고 "진행 중"인 작업을 나타낼 때 사용됩니다.

**Success** — 성공적으로 완료된 프로세스 또는 엔티티의 "상태"를 표시할 때 사용됩니다.

**Warning** — 비정상적인 상황에서 사용자에게 무언가를 경고할 때 사용됩니다.

**Danger** — 오류가 발생하여 문제를 해결하기 위한 조치가 필요할 때 최대한의 주의를 끌기 위해 사용됩니다.

### 의미론적 레이블 vs 버튼

의미론적 색상이 있는 레이블을 포함한 레이블은 상태를 나타내는 데 사용됩니다. 레이블과 상호작용할 수 있는 유일한 방법은 레이블에 마우스를 올리거나 아이콘을 클릭하는 것입니다. 레이블은 `Selected` 상태를 가질 수 없습니다. 의미론적 레이블을 기반으로 인터랙티브 요소를 만들어야 하는 경우(예: 상태별 필터링), 버튼을 사용하는 것이 좋습니다. 버튼은 눌린(적용된) `Selected` 상태와 의미론적 색상 지원을 제공합니다.

### 수정 및 상태

모든 레이블은 5가지 상태를 가집니다: 텍스트만, 텍스트 앞 아이콘, 텍스트 뒤 아이콘, 앞뒤 아이콘, 아이콘만.

![States](/static/images/design/Label/States.png)

![States](/static/images/design/Label/States2.png)

상호작용할 수 있는 부분:

1. 전체 레이블 — 예를 들어, 레이블에 마우스를 올리면 자세한 정보가 포함된 툴팁이 나타납니다.

2. 별도의 오른쪽 아이콘 — 아이콘에 따라 클릭하면 레이블을 삭제하거나 해당 값을 복사할 수 있습니다.

레이블에 상호작용을 추가할 때의 주요 규칙은 사용자가 레이블과의 상호작용 가능성을 이해하도록 하는 것입니다.

### 추가 기능

레이블을 기반으로 콘텐츠 정렬을 위한 동적 필터를 만들 수 있습니다. 더하기 아이콘을 클릭하면 새 값이 추가되고, 닫기 아이콘을 클릭하면 필터가 취소됩니다.

![Features](/static/images/design/Label/Features.png)

인터페이스에서 사용자가 자체 레이블을 만들고 객체를 표시할 수 있다면, 사용자는 레이블을 추가하고 삭제할 수 있어야 합니다.

### 사용자 정의

레이블의 색상, 둥근 모서리, 크기 및 간격을 변경할 수 있습니다. 레이블의 텍스트는 쉽게 읽을 수 있어야 합니다.

![Customization](/static/images/design/Label/Customization.png)

사용자가 레이블에 마우스를 올리면, 상호작용 가능성을 알리기 위해 모양이나 색상을 변경하십시오.
