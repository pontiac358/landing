Hier ist die Übersetzung der Design-Guide-Quelldatei ins Deutsche, die versucht, so natürlich wie möglich zu klingen und dabei alle MDX-Komponenten-Tags, Codeblöcke und technischen Begriffe beizubehalten:

Klickbares Element mit Navigationsfunktion. Beim Klicken öffnet es eine andere Seite eines externen Dienstes oder innerhalb des Dienstes. Seltener bringt es den Benutzer zu einem anderen Teil der Seite.

Es kann drei Typen geben: `normal` (akzentuiert), `primary` (schwarz), `secondary` (grau).

**Typ `normal`**

Das gebräuchlichste und etablierteste Nutzungsmuster für Benutzer. Wird verwendet, um ein Element innerhalb von Text, Tabellen visuell abzugrenzen und als Teil der Navigation. Wird auch auf Fehlerseiten und Nullzuständen verwendet.

**Typ `primary`**

Wird verwendet, wenn für den Benutzer intuitiv klar ist, dass das Element interaktiv ist, aber die Verwendung von Links in Akzentfarbe die Benutzeroberfläche überladen und das Gleichgewicht der Elemente auf der Seite stören würde.

**Typ `secondary`**

Ähnlich wie `primary`-Links werden sie verwendet, wenn für den Benutzer klar ist, dass das Element interaktiv ist, und es die Aufgabe gibt, nicht mit den Akzenten auf der Seite zu konkurrieren. Wird häufiger in Breadcrumbs oder bei der Anzeige von sekundären Attributen verwendet.

## **Zustände**

Links haben immer einen Standard- und einen Hover-Zustand.

![Frame 507123500.jpg](/static/images/design/Links/Frame_507123500.png)

Bei Bedarf kann für besuchte Links eine Markierung aktiviert werden.

## **Können verwendet werden als:**

- eigenständige Elemente;
- Teil eines zusammenhängenden Textes.

Sie können ein Symbol als Indikator für die Weiterleitung zu einer externen Ressource haben.

![Frame 507123501.jpg](/static/images/design/Links/Frame_507123501.png)

## Größe

Die Schriftgröße für Links sollte der allgemeinen Hierarchie auf der Seite entsprechen und proportional zum umgebenden Text sein.

## Empfehlungen

Der Linktext sollte für den Benutzer auch unabhängig vom restlichen Text verständlich sein und Informationen darüber enthalten, wohin der Link weiterleitet (Name des Dienstes, Titel oder Beschreibung des Abschnitts).
