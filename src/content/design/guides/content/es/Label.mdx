El `Label` contiene información textual sobre un objeto y describe una de sus propiedades, como el estado o el tipo. Los `Label` se utilizan para categorizar, marcar o clasificar, así como para etiquetar entidades en un formato de "clave-valor".

### Aplicación

Los `Label` se utilizan de forma generalizada (por ejemplo, en tarjetas, tablas, páginas de entidades) para indicar las propiedades de un objeto y ayudan a los usuarios a identificar y clasificar el contenido rápidamente. Normalmente, los `Label` se muestran junto a los objetos a los que se refieren.

Una aplicación específica es mostrar la cantidad de objetos que no caben en una lista. En este caso, al pasar el cursor sobre el `Label`, aparece una ventana emergente (`popover`) con la lista de las entidades restantes.

![DLQ](/static/images/design/Label/DLQ1.png)

Los `Label` también pueden utilizarse como etiquetas para organizar entidades y facilitar su manejo.

![DLQ](/static/images/design/Label/DLQ2.png)

Al mostrar `Label` en una tabla, se debe limitar la cantidad de `Label` por celda (un máximo de 2-3 `Label` seguidos y un máximo de 2 columnas de `Label` en una misma tabla), así como el número máximo de caracteres (un máximo de 20 caracteres por `Label`, seguido de puntos suspensivos).

### Tamaños

![Sizes](/static/images/design/Label/Sizes.png)

**Existen 3 tamaños de etiqueta:**

**XS** — se utiliza en la mayoría de los lugares: tablas, listas, encabezados, pop-ups, página de entidad.

**S** — se utiliza en formularios para previsualizar etiquetas añadidas y donde el espacio lo permite.

**M** — un tamaño similar se utiliza con menos frecuencia porque se asemeja a un botón.

### Temas

![Themes](/static/images/design/Label/Themes.png)

![Themes](/static/images/design/Label/Themes2.png)

**Sin tinte semántico**:

**Default** — etiqueta por defecto, a menudo utilizada para resaltar información, sin tinte semántico.

**Clear** — no tiene fondo, solo contorno. Se utiliza para un acento mínimo.

**Con tinte semántico**:

**Unknown** — variante neutral, se utiliza cuando no se requiere acentuar la información envuelta en la etiqueta.

**Accent** — para resaltar información importante y para operaciones "en curso".

**Success** — para mostrar un proceso completado con éxito o la "salud" de una entidad.

**Warning** — para situaciones anómalas, para advertir al usuario sobre algo.

**Danger** — para atraer la máxima atención cuando ha ocurrido algún error y se necesitan tomar acciones para solucionar el problema.

### Etiquetas semánticas vs. botones

Las etiquetas, incluidas las de tinte semántico, se utilizan para indicar un estado. La única interacción disponible con ellas es el `hover` sobre la etiqueta o el clic en el icono. La etiqueta no puede estar en el estado `Selected`. Si necesitas crear un elemento interactivo basado en etiquetas semánticas (por ejemplo, para filtros por estado), es mejor usar botones: estos tienen un estado presionado (aplicado) `Selected` y soporte para colores semánticos.

### Funcionalidades adicionales

Basándose en la etiqueta, se pueden crear filtros dinámicos para ordenar el contenido. Al hacer clic en el icono con el signo más se añade un nuevo valor, y al hacer clic en la cruz se cancela el filtro.

![Features](/static/images/design/Label/Features.png)

Si tu interfaz permite a los usuarios crear sus propias etiquetas y marcar objetos, los usuarios deben tener la posibilidad de añadir y eliminar etiquetas.

### Personalización

En una etiqueta se pueden cambiar los colores, el redondeo, los tamaños y los márgenes. El texto de la etiqueta debe ser fácilmente legible.

![Customization](/static/images/design/Label/Customization.png)

Cuando el usuario pasa el cursor del ratón sobre la etiqueta, cambia su apariencia o color para indicar que es interactiva.
