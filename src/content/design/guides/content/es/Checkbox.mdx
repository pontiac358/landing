El `Checkbox` es un elemento de interfaz que permite a los usuarios seleccionar una o varias opciones de una lista.

## Casos de uso

![Cases](/static/images/design/Checkbox/Checkbox1.png)

Después de activar un `Checkbox`, la acción no se ejecuta de inmediato. Para que los cambios surtan efecto, es necesario hacer clic en un botón de confirmación (por ejemplo, "Guardar", "Aplicar", "Enviar"). En la mayoría de los casos, el `Checkbox` se utiliza junto con una etiqueta que explica su propósito. Sin embargo, hay escenarios en los que un `Checkbox` puede usarse sin etiqueta, por ejemplo, en tablas o cuando mostrar la etiqueta "Habilitar" junto a cada `Checkbox` podría ser redundante. El texto de la etiqueta debe comenzar con una formulación positiva, como "Habilitar" en lugar de "Deshabilitar", de lo contrario, el usuario tiene que resolver un problema lógico.

## Anatomía

Un `Checkbox` consta de:

![Anatomy](/static/images/design/Checkbox/Checkbox2.png)

1.  **Contenedor**: un área visual de forma rectangular que rodea el `Checkbox`. El contenedor sirve para delimitar y activar el `Checkbox`.
2.  **Marca de verificación**: un símbolo que aparece dentro del contenedor cuando el `Checkbox` está activado. Este símbolo sirve como confirmación visual de que el usuario ha seleccionado este `Checkbox`.
3.  **Etiqueta de texto (opcional)**: texto que se coloca a la derecha del `Checkbox`. La etiqueta de texto explica el significado o el propósito del `Checkbox`.

## Tamaños

El componente `Checkbox` tiene dos tamaños principales:

![Sizes](/static/images/design/Checkbox/Checkbox3.png)

**M**: se utiliza por defecto y tiene tamaños estándar adecuados para la mayoría de los casos. En este tamaño, el contenedor es de 14x14 px y para la etiqueta de texto se utiliza una fuente de 13 px.

**L**: se utiliza si es necesario destacar el `Checkbox` entre otros elementos de la interfaz o si la interfaz requiere elementos más grandes para facilitar su uso. En este tamaño, el contenedor es de 17x17 px y para la etiqueta de texto se utiliza una fuente de 15 px.

## Estados

![States](/static/images/design/Checkbox/Checkbox4.png)

Aquí tienes la traducción del archivo fuente de la guía de diseño al español, manteniendo un tono natural y conservando todos los componentes MDX, bloques de código y términos técnicos:

**Predeterminado** — el estado principal de una casilla de verificación, que se puede activar o desactivar haciendo clic.

**Hover** — el estado cuando el cursor del ratón se cierne sobre el área de la casilla de verificación (casilla de verificación + etiqueta).

**Deshabilitado** — el estado cuando la interacción con la casilla de verificación no está disponible por alguna razón.

**Indeterminado** — el estado del encabezado de un grupo de casillas de verificación cuando se selecciona uno de varios elementos (se encuentra a menudo en tablas con selección múltiple de filas).

## Recomendaciones

### Cuándo usar

- **Selección de opciones** — cuando el usuario necesita seleccionar una o varias opciones de una lista presentada.

- **Activación/desactivación de funciones** — cuando el usuario puede activar o desactivar ciertas funciones en la interfaz. Por ejemplo, se pueden activar o desactivar notificaciones, reproducción automática y otras funciones. En estos casos, las casillas de verificación proporcionan una interacción intuitiva.

- **Selección múltiple en tablas o listas** — cuando el usuario necesita seleccionar varios elementos para operaciones grupales, como eliminar o exportar. Esto simplifica el trabajo con los datos y hace que la ejecución de acciones grupales sea más conveniente.

- **Filtrado de contenido o resultados** — en filtros, donde el usuario selecciona parámetros para filtrar contenido o resultados de búsqueda.

### Cuándo no usar

- **Selección de una sola opción** — si el usuario debe seleccionar solo una opción de un conjunto de variantes mutuamente excluyentes, es mejor usar botones de radio (radio).

- **Activación/desactivación de dos posiciones (acción instantánea)** — si una opción solo tiene dos estados (activado y desactivado) y los cambios deben aplicarse instantáneamente sin hacer clic en el botón "Aplicar", entonces es mejor usar interruptores (switches) para este propósito. Proporcionan una representación más clara y concisa para tales opciones y permiten aplicar los cambios de inmediato.

- **Selección segmentada** — Si hay varias opciones mutuamente excluyentes que se pueden seleccionar simultáneamente en diferentes categorías, utiliza controles segmentados (segment controls). Estos permiten al usuario seleccionar opciones en diferentes segmentos o categorías.

- **Selección de un gran número de opciones** — Si hay una lista larga de opciones de las que se debe elegir una, es mejor usar un menú desplegable (select). Esto ahorrará espacio en la pantalla y facilitará la selección entre un gran conjunto de opciones.

### Redacción de textos

- Busca la brevedad en la descripción del texto. Evita frases largas o construcciones complejas.
- Utiliza verbos activos que indiquen claramente la acción relacionada con la selección (por ejemplo, "Suscribirse al boletín" en lugar de "Suscripción al boletín").
- Ofrece a los usuarios una idea completa de lo que sucederá si seleccionan una casilla de verificación (por ejemplo, "Recibir notificaciones sobre nuevos artículos" en lugar de "Notificaciones").
